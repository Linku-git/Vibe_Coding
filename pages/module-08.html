Using Google Stitch for Design-to-Code

Integrate AI-generated designs into your development workflow using Google Stitch.

What is Google Stitch?

Google Stitch is a free, AI-powered design tool from Google Labs that uses Gemini to generate high-fidelity UI screens from plain English descriptions or reference images. It bridges the gap between "I have an idea" and "here's a working UI" — and it outputs a DESIGN.md file that AI coding tools can consume directly.

Why Stitch Matters for Vibe Coding

Without a design reference, AI generates inconsistent UIs — different colors on each page, mismatched spacing, random layouts. Stitch gives your AI agent a single source of design truth: a DESIGN.md file with exact tokens, components, and rules to follow.

Two Integration Methods

Method 1: MCP Connection (Recommended)

Connect Stitch directly to your AI coding tool via MCP. The agent can pull designs, extract tokens, and generate matching code — all without manual file management.

1
Enable the Stitch API in Google Cloud

Go to Google Cloud Console, enable the Stitch API for your project.

2
Configure local authentication
gcloud auth application-default login
3
Add the MCP server to your AI tool
.claude/settings.json — Stitch MCP
{
  "mcpServers": {
    "stitch": {
      "command": "npx",
      "args": ["-y", "stitch-mcp"],
      "env": {
        "GOOGLE_CLOUD_PROJECT": "your-project-id"
      }
    }
  }
}

Available MCP Tools:

  • list_projects — Browse your Stitch projects
  • list_screens — List all screens in a project
  • extract_design_context — Scan screens and extract design tokens (colors, fonts, spacing)
  • fetch_screen_code — Download the raw HTML/CSS/React code for a screen
  • fetch_screen_image — Download high-resolution screenshots
  • generate_screen_from_text — Create new screens from text prompts
Claude Code Workflow: Design screens in Stitch using natural language → Stitch MCP pulls the DESIGN.md and screen code → Claude Code generates production code that matches the design exactly → Consistent UI across the entire app.
Method 2: ZIP Export + Design Folder

If you prefer manual control or don't want MCP, export your Stitch project as a ZIP file and add it to your repo.

1
Export from Stitch

In Stitch, click Export → Download as ZIP. This gives you HTML files and image assets.

2
Extract to a /design folder in your project
3
Reference the design folder in CLAUDE.md

Add a section telling the AI to read design/DESIGN.md before generating any UI code.

CLAUDE.md Addition for Design Reference

Add to CLAUDE.md
## Design System
- ALWAYS read design/DESIGN.md before creating or modifying UI
- Match colors, typography, and spacing from design tokens exactly
- Use the component patterns from design/components/ as reference
- Check design/screens/ for the intended layout of each page
- Do not invent new colors or spacing — use only defined tokens
- If a screen is not in the design folder, ask before implementing

Best Practices

Lock design tokens early. Decide on your colors, typography, and spacing before generating additional screens. Changing tokens later means regenerating everything.
  • Iterate in Stitch first, then hand to code AI. It's faster to fix a layout in Stitch than to rewrite generated code.
  • Treat Stitch output as a foundation, not finished code. Clean up, integrate with your component library, and apply your conventions.
  • Extract reusable components. Don't re-export the same button 20 times. Extract it once and reference it.
  • Use specific prompts in Stitch. "A dashboard with a sidebar nav, 4 KPI cards, a data table, and a line chart" beats "a dashboard."
  • Version your design folder. Commit /design to git so the AI always has the latest approved designs.
