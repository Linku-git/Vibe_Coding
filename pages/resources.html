CLAUDE.md
# My Project — Agent Instructions

## Project Overview
Read Docs/prd.md for full requirements.

## Stack
- Frontend: [framework]
- Backend: [framework]
- Database: [database]

## Development Workflow
- ALWAYS read Docs/prd.md before implementing
- Work only on the current session (check Docs/sessions/)
- Make small, atomic, testable changes
- Never implement features from future sessions

## After Every Session
Update these files:
- Docs/progress.md — mark completed tasks
- Docs/sessions/session-XX.md — add implementation notes
- Docs/api-routes.md — if backend routes changed
- Docs/frontend-pages.md — if UI changed

## Testing
- Run tests after every change
- Document test results in session file
- Do not mark tasks complete until tests pass

## Design System
- Read design/DESIGN.md before creating or modifying UI
- Match colors, typography, and spacing from design tokens
- Do not invent new styles — use defined tokens only

## MCP Tools
- GitHub MCP: for PRs and issues only
- PostgreSQL MCP: for DB queries and schema inspection
- Playwright MCP: for UI testing only

- [Add your project conventions here]
