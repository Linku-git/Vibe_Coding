agents.md Structure (for Codex / Antigravity)

Docs/agents.md
# AI Development Agent Rules

## 1. General Workflow
Read PRD first.
Do not skip architecture.
Work in small increments.

## 2. Session Rules
One session at a time.
Do not merge multiple major features in one session.

## 3. Documentation Rules
Update progress, readme, routes, pages, and session
logs after each session.

## 4. Testing Rules
Run relevant tests after every session.
Document exact commands and results.

## 5. Error Reporting
Log all errors, failed tests, root causes, and fixes.

## 6. Completion Rules
A task is complete only if implementation, local run,
and documentation are done.

CLAUDE.md Structure (for Claude Code)

Claude Code uses CLAUDE.md instead of (or in addition to) agents.md. It's read automatically.

CLAUDE.md
# My Project — Agent Instructions

## Project Overview
Read Docs/prd.md for full requirements.

## Development Workflow
- ALWAYS read Docs/prd.md before implementing
- Work only on the current session (check Docs/sessions/)
- Make small, atomic, testable changes
- Never implement features from future sessions

## After Every Session
Update these files:
- Docs/progress.md — mark completed tasks
- Docs/sessions/session-XX.md — add implementation notes
- Docs/api-routes.md — if backend routes changed
- Docs/frontend-pages.md — if UI changed
- Docs/readme.md — if setup steps changed

## Testing
- Run `npm test` and `pytest` after every change
- Document test results in session file
- Do not mark tasks complete until tests pass

## Error Handling
- Log all errors with root cause in session file
- Fix before proceeding to next task
- If blocked, document the blocker clearly

## Conventions
- TypeScript strict mode for frontend
- Pydantic models for all API request/response schemas
- All forms must validate client-side and server-side
- Use meaningful commit messages
