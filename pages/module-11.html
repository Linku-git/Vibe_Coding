Minimum Testing Loop

1
Run the app
2
Check the feature manually
3
Run automated tests
4
Check console and terminal errors
5
Update docs with results

Frontend Checks

  • Page loads
  • No console errors
  • Forms validate
  • Buttons work
  • Navigation works
  • Empty states work
  • Error states work

Backend Checks

  • Server starts
  • Routes respond
  • Bad input rejected
  • DB reads/writes work
  • Auth/permissions work
  • No major log errors

Full-Stack Checks

  • Frontend-backend connection
  • API responses correct
  • Auth flow end-to-end
  • CRUD flow complete
  • Validation flow works
  • Edge cases handled

Claude Code Testing Integration

Claude Code can run tests directly in your terminal. Use hooks for automated testing:

.claude/settings.json — hooks example
{
  "hooks": {
    "postToolUse": [
      {
        "matcher": "Edit|Write",
        "command": "npm run lint --quiet"
      }
    ]
  }
}

This automatically lints after every file edit, catching issues immediately.
