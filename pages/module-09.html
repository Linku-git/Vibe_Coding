Creating Coding Sessions

Split the entire app into small, executable, testable sessions.

Video coming soon

Why Sessions Matter

Without sessions, AI tries to build too much at once. With sessions: work stays focused, testing is easier, bugs are easier to isolate, and documentation stays current.

Session Rules

A session should be: small, atomic, testable, reviewable, and independent enough to complete safely.

How many sessions? Simple projects need 6-10 sessions. Complex full-stack projects need 15-30+ sessions. The number depends on project scope — don't force a complex app into 8 sessions. Each session should be completable in one AI conversation.

Example Session Breakdown — Complex Project (15 sessions)

Phase 1: Foundation
01Project bootstrap, folder structure, and dev environment
02Database models, migrations, and seed data
03Authentication backend (routes, middleware, tokens)
Phase 2: Core Features
04Authentication frontend (login, register, protected routes)
05Dashboard shell, navigation, and layout
06Primary feature CRUD — backend routes and logic
07Primary feature CRUD — frontend pages and forms
08Secondary feature implementation
Phase 3: Advanced Features
09Search, filtering, and pagination
10File uploads or media handling
11Notifications or real-time features
Phase 4: Integration & Polish
12Validation, error states, and edge cases
13Responsive design and accessibility
14Performance optimization and caching
Phase 5: Testing & Deployment
15End-to-end tests, cleanup, and deployment prep

# Session 01 — Project Bootstrap
## Phase: Foundation

## Goals
Set up initial project structure and development environment.

## Tasks
- [ ] Initialize repo
- [ ] Create Docs structure
- [ ] Scaffold frontend
- [ ] Scaffold backend
- [ ] Create env example
- [ ] Add formatter/linter
- [ ] Add base test setup

## Files to Create/Update
- package.json
- requirements.txt
- CLAUDE.md
- .gitignore
- Docs/progress.md

## Implementation Notes
(filled in during/after session)

## Tests
- npm install && npm run dev
- pytest
- npm run lint

Project runs locally with no errors.
