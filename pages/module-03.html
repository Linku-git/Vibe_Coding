Setup Steps

1
Create a fresh folder and initialize Git
mkdir my-project && cd my-project && git init
2
Create docs structure
mkdir -p Docs/sessions
3
Create base files
touch Docs/prd.md Docs/agents.md Docs/progress.md Docs/readme.md
touch .env.example .gitignore CLAUDE.md
4
Install runtime and dependencies for your stack

React: npm init / Flask: python -m venv .venv

5
Set up linting, formatting, and tests early

Critical — AI code becomes harder to control if validation comes too late.

Claude Code Setup: CLAUDE.md

Claude Code uses CLAUDE.md as project-level instructions. Place it in your project root:

CLAUDE.md
# Project: My App

## Stack
- Frontend: React + TypeScript
- Backend: Flask + Python 3.12
- Database: SQLite (local), PostgreSQL (prod)

## Development Rules
- Always read Docs/prd.md before implementing features
- Work in atomic sessions defined in Docs/sessions/
- Run tests after every change
- Update Docs/progress.md after every session

## Conventions
- Use React Query for server state
- All backend routes must have schema validation
- All forms must have validation and clear error messages
- Use SQLite during local development

## Commands
- Frontend dev: `cd frontend && npm run dev`
- Backend dev: `cd backend && flask run`
- Tests: `npm test` / `pytest`
- Lint: `npm run lint` / `ruff check .`

Important Rule

Before asking AI to code, you should already have:

  • Folder structure
  • Stack choice
  • prd.md
  • agents.md / CLAUDE.md
  • Base session plan
  • Run/test commands
