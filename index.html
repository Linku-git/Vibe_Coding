What is Vibe Coding?

Vibe coding is not "ask AI to build everything blindly." It's a disciplined, documentation-first process that makes AI your most effective team member.

✗ Random AI Coding

  • One giant prompt: "build me an app"
  • No documentation
  • No testing between changes
  • Messy folder structure
  • AI invents architecture
  • Impossible to continue later

✓ Vibe Coding Process

  • Define the project clearly in a PRD
  • Structure the repo before coding
  • Give AI documentation and rules
  • Break work into small sessions
  • Run & test after every session
  • Update docs after every session

Key Lesson

AI works best when you give it: context, constraints, structure, and checkpoints.

Key Concepts & Tools Explained

Understand the building blocks that make AI-assisted development work.

Skills

Skills are plain markdown files that teach the AI how to perform a specific task. They live in .claude/commands/ and require no server, API, or dependencies.

Examples

  • /project:run-session 03 — execute a coding session
  • /project:add-api-route — add a backend route with validation and tests
  • /project:review-code — scan for security and performance issues

Agents

An agent is a set of behavioral rules that define how the AI approaches work — what to read first, when to test, what to update.

Examples

  • Research agent: "Explore the codebase — report only, change nothing"
  • Implementation agent: "Implement session-06, follow the PRD, run tests"
  • Testing agent: "Run all tests, classify failures, suggest fixes"

MCP Servers

MCP (Model Context Protocol) connects AI agents to external tools and data. Without MCP, the AI can only read files and run commands. With MCP, it can query databases, create PRs, pull designs, and more.

Examples

  • GitHub MCP — create PRs, manage issues
  • PostgreSQL MCP — query database, inspect schema
  • Playwright MCP — browser automation, screenshots
  • Stitch MCP — pull UI designs from Google Stitch

Memory

Memory is the AI's persistent context layer. It remembers decisions, preferences, and conventions across conversations.

Good Memory Items

  • "Use React Query for server state"
  • "All API routes must have Pydantic validation"
  • "Deployment target is Vercel"
When Something Fails or Requires Manual Setup — Always Ask AI How

AI agents can't do everything. Some steps require manual action from you. That's normal.

If the AI can't do it for you, ask the AI to tell you exactly how to do it yourself.

Never get stuck silently. The AI is your guide even when it can't be your hands.

How It All Fits Together

Agents define the rules. Skills encode the procedures. MCP Servers provide the connections. Memory maintains context. Tools provide the infrastructure. Together, they turn AI into a disciplined team member.
